# 退款政策记录

> 用途：客服回复退款问题时的事实依据。

## 回复原则

- 不由 AI 直接承诺退款结果。
- 先安抚用户，再收集信息。
- 转人工处理。
- 不与用户争辩责任归属。
- 不在强烈不满场景继续销售转化。

## 需收集信息

- 账号或用户名。
- 购买时间。
- 支付金额。
- 订单号或支付截图。
- 退款原因。
- 遇到的问题截图或描述。

## 标准回复

```text
很抱歉给你带来了不好的体验。
退款问题需要人工客服进一步核实处理。

请提供：
1. 账号或用户名
2. 购买时间和金额
3. 订单截图
4. 退款原因

我会帮你转接人工客服。
```

## 禁止

- 不承诺“马上退款”。
- 不承诺“一定能退”。
- 不要求用户在公开渠道发送订单截图。
- 不让模型自行解释复杂退款政策。
