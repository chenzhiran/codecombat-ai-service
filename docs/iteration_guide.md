# 迭代优化指南

> 适用于 CodeCombat 中国智能客服系统的日常维护、质量审计和模型升级。

## 每日

- 导出未识别、低置信度、转人工失败的对话。
- 同一问题频次 ≥ 3 次 / 天，当天新建或补充 intent。
- 检查人设一致性异常：安雅不能出现扣哒鸭语气，扣哒鸭不能自称安雅。
- 检查禁忌触发：价格编造、竞品攻击、承诺就业、泄露内部信息。
- 记录需要人工补充事实的问答，进入本周知识库更新清单。

## 每周

- 补充新增高频问题（≥ 5 次 / 周）。
- 从真实对话中提取新的相似问法，补到对应 `sample_questions`。
- 检查链接、价格、功能说明、购买入口是否过期。
- 抽检至少 50 条对话，审计人设、事实、转人工和用户体验。
- 同步官网 FAQ 和运营活动信息。
- 对 evaluation 中的测试用例做一次回归。

## 每月

- 意图体系评审：新增、合并、拆分或废弃 intent。
- 扣哒鸭小红书内容方向复盘：互动率、私信转化、负面反馈。
- 用户故事库更新：补充自学者、转行者、家长案例。
- 竞品客服体验对标，但不在回复中攻击竞品。
- 检查 system prompt 是否过长，拆分为可检索知识。

## 每季度

- 全量知识库审计。
- 新功能、新价格、新政策补充。
- 核心指标复盘：准确率、满意度、自助解决率、转化率。
- 人设迭代评估：是否需要调整语气、禁忌、渠道边界。
- 模型能力复评：上下文窗口、结构化输出、工具调用、中文稳定性、延迟和成本。

## 2026 模型升级检查项

- Router 是否输出结构化字段，而不是只输出自然语言判断。
- 是否按用户 intent 检索相关文档，避免一次加载全部知识。
- 是否记录 `intent_id`、`agent`、`persona`、`confidence` 和 `requires_human`。
- 是否有自动评测集覆盖官网、小红书、关卡 AI 三个核心场景。
- 是否把价格、政策、优惠等高风险内容放到可审计文档中。

## 修改流程

1. 先定位问题属于 persona、agent、skill、workflow、knowledge base、prompt 还是 evaluation。
2. 修改最小必要文档，不把临时活动写入长期规则。
3. 同步更新相关 few-shot 示例和测试用例。
4. 用 evaluation 文档回归核心场景。
5. 记录修改原因、日期和负责人。
