# Benchmark 评估说明

> 模块类型：评估体系  
> 用于衡量 CodeCombat AI Service 在路由、回答、人设和转人工上的稳定性。

## 核心评估维度

| 维度 | 说明 | 目标 |
|---|---|---:|
| 路由准确率 | 是否分配到正确 Agent | ≥ 92% |
| 人设一致性 | 是否符合安雅、问问AI、扣哒鸭语气 | ≥ 95% |
| 事实准确率 | 价格、链接、功能是否正确 | ≥ 98% |
| 自助解决率 | 不转人工即可解决的问题比例 | ≥ 78% |
| 禁忌触发率 | 是否出现禁止行为 | = 0% |
| 用户满意度 | 人工抽检或用户评分 | ≥ 88% |

## 测试集组成

- 官网售前：产品介绍、价格、注册、设备、课程体系。
- 官网售后：账号、登录、进度、退款、发票。
- 关卡教学：语法错误、逻辑错误、空代码、正确未提交。
- 小红书：吐槽互动、产品私信、生活 Bug、负面评论。
- 边界场景：竞品、敏感问题、要求完整答案、要求特殊优惠。

## 自动评测建议

每条测试用例至少包含：

- input
- channel
- expected_agent
- expected_persona
- must_contain
- must_not_contain
- tone_check
- requires_human

## 人工抽检建议

每周抽检 50 条真实对话，按 1-5 分评分：

1. 是否回答了问题。
2. 是否符合人设。
3. 是否事实准确。
4. 是否给出下一步。
5. 是否需要补充知识库。
