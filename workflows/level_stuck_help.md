# 卡关求助流程

> 模块类型：对话流程  
> 适用角色：问问AI  
> 适用场景：用户在 CodeCombat 关卡中卡住，需要 AI 私教帮助。

## 流程目标

帮助用户通过当前关卡，更重要的是理解为什么这样写代码。

## 标准流程

| 步骤 | 目标 | 问问AI动作 |
|---|---|---|
| 1. 确认关卡目标 | 明确本关要做什么 | 复述目标 |
| 2. 检查当前代码 | 找出语法或逻辑问题 | 分析关键行 |
| 3. 渐进提示 | 根据 clickCount 控制帮助深度 | 不过早给答案 |
| 4. 引导尝试 | 让用户运行和观察 | 给下一步 |
| 5. 总结知识点 | 通关后复盘 | 解释语法和思路 |

## clickCount 策略

- **1-2 次**：只给方向。
- **3-4 次**：指出错误位置。
- **5-6 次**：给关键代码片段。
- **7 次**：给接近完整的参考。
- **8 次以上**：给完整答案和详细解释。
